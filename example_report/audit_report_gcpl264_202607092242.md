### 📉 Excessive Dynamic Fields (Table Calculations / Custom Fields)
**Finding**: Dashboard ID **14** ("Google PSO - Performance Metrics") contains queries with up to **15 dynamic fields**.
*   **Impact**: Bypasses LookML governance, strains browser memory, causes dashboard lag.

**🛠️ Action**: Move complex calculations into LookML as dimensions or measures.
*   **Command to identify specific tiles/queries on Dashboard 14**:
    ```bash
    echo '{"model":"system__activity","view":"dashboard","fields":["dashboard.id","dashboard.title","query.id","query.count_of_dynamic_fields"],"filters":{"dashboard.id":"14","query.count_of_dynamic_fields":">2"},"limit":"50"}' | looker-cli api query run_inline_query json - | jq
    ```
*   **Example LookML Fix**:
    ```lookml
    # Move from Table Calc to LookML Dimension
    dimension: profit {
      type: number
      sql: ${sale_price} - ${cost} ;;
      value_format_name: usd
    }
    ```

### 🔑 Missing Primary Keys
**Finding**: Multiple views in project `mtr_abraham_thelook` are missing primary keys.
*   **Examples**: `PDT_test.view.lkml`, `explores/test.view.lkml`.
*   **Impact**: Risk of incorrect symmetric aggregates, join issues.

**🛠️ Action**: Define `primary_key: yes` on the unique identifier column in each of these views.
*   **Command to verify missing primary keys in a project**:
    ```bash
    for file in $(looker-cli api project all_project_files mtr_abraham_thelook | jq -r '.[] | select(.type=="view") | .path'); do   looker-cli project file cat mtr_abraham_thelook "$file" | grep -q "primary_key: yes" || echo "Missing primary key in $file"; done
    ```
*   **Example LookML Fix**:
    ```lookml
    dimension: id {
      primary_key: yes
      type: number
      sql: ${TABLE}.id ;;
    }
    ```
